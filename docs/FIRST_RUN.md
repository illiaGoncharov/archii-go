# Первый запуск без Firebase

## Зачем этот документ

Firebase **НЕ обязателен** для первичного запуска и проверки UI. Можно сначала убедиться, что всё работает, а Firebase настроить позже для тестирования функциональности.

---

## Что работает без Firebase

- Навигация между экранами
- UI компоненты (карточки, кнопки, модальные окна)
- Дизайн-система (цвета, типографика, spacing)
- Конструктор фасадов (визуально)
- Tab navigation
- Экран карты (визуально, без данных)

## Что НЕ работает без Firebase

- Загрузка данных (элементы, POI)
- Авторизация
- Сохранение находок
- Загрузка фото
- Работа с коллекцией (пустая)

---

## Порядок действий

### 1. Установка (2 минуты)

```bash
npm install
```

Должен установить все зависимости без ошибок.

### 2. Запуск на веб (для проверки UI)

```bash
npm run web
```

**Ожидаемое:**
- Откроется в браузере (http://localhost:8081)
- Увидишь табы: Карта, Камера, Коллекция, Профиль
- Можешь переключаться между экранами
- UI отрисовывается (даже если данные не загружаются)

**Что проверить:**
- Навигация работает
- Компоненты отрисовываются
- Нет красных ошибок в консоли
- Табы переключаются

**Если ошибки:**
- Проверь консоль браузера (F12)
- Ошибки про Firebase — нормально (пока не настроен)
- Другие ошибки — сообщи, разберёмся

### 3. Запуск на iPhone (через Expo Go)

```bash
npm start
```

**Действия:**
1. Установи Expo Go из App Store (если ещё не установлен)
2. Отсканируй QR-код из терминала камерой iPhone
3. Приложение откроется в Expo Go

**Что проверить:**
- Все экраны открываются
- Навигация работает
- UI выглядит нормально
- Разрешения на камеру/геолокацию запрашиваются

**Что НЕ будет работать (без Firebase):**
- Данные в коллекции (пустая)
- POI на карте (нет)
- Сохранение находок (ошибка)

---

## Когда нужен Firebase

Настрой Firebase, когда захочешь протестировать:
- Загрузку элементов из базы
- Создание находок
- Сохранение фото
- Авторизацию
- Работу с коллекцией

**Инструкция:** [SETUP.md](SETUP.md)

## Мокирование данных (опционально)

Если хочешь посмотреть на UI с данными, можно временно замокировать:

### В `app/(tabs)/collection.tsx`:

```typescript
// Временно, для теста
const mockElements = [
  {
    id: '1',
    name: 'Метопа',
    description: 'Тестовый элемент',
    style: 'Классицизм',
    rarity: 'common',
    points: 10,
    imageUrl: 'https://via.placeholder.com/300',
    totalFound: 0,
    isFound: false,
  },
  // ... ещё элементы
];

// Используй mockElements вместо userElements
```

**Но лучше:** настроить Firebase один раз и работать с реальными данными.

## Итого

**Сначала:** проверь UI без Firebase  
**Потом:** настрои Firebase для функциональности

Это быстрее и логичнее, чем настраивать Firebase сразу.
