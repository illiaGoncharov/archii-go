# Дизайн-манифест Archi-Go

## "Структурная эстетика"

Archi-Go следует философии **"Структурной эстетики"** — дизайн-подходу, где цифровой интерфейс строится по тем же принципам, что и архитектура: точность пропорций, модульность, чистота линий, уважение к деталям.

---

## Философия

### 1. Чертёж как вдохновение

Интерфейс подобен архитектурному чертежу:
- **Чистый фон** — как бумага для чертежа, не конкурирует с контентом
- **Чёткая сетка** — модульная структура, как у зданий
- **Точные пропорции** — всё на своих местах, как в хорошем фасаде
- **Приглушённые цвета** — акценты только там, где нужно внимание

### 2. Детали архитектуры — звёзды шоу

UI не должен затмевать архитектурные элементы:
- **Нейтральная палитра** — чтобы метопы, сандрики и капители выделялись
- **Воздух вокруг** — достаточно пространства, чтобы разглядеть детали
- **Контрастность** — элементы читаются чётко на любом фоне

### 3. Типографика как конструкция

Текст — это тоже архитектура:
- **IBM Plex** — технический, но гуманный. Как хороший модернизм
- **Меньшие размеры** — плотность без суеты
- **Чёткая иерархия** — заголовки как колонны, текст как кладка

### 4. Цвет как акцент, не украшение

Цвет работает, а не развлекает:
- **Монохром по умолчанию** — спокойная серая шкала
- **Акценты для действий** — синий для кликабельного
- **Цвета редкости** — приглушённые, но различимые
- **Семантика стилей** — каждому архитектурному стилю свой оттенок

### 5. Структура, а не декорация

Каждый элемент на своём месте:
- **Модульная сетка** — как несущая конструкция
- **Последовательность** — одинаковые отступы, радиусы, паттерны
- **Функциональность** — кнопка выглядит как кнопка, карточка как карточка
- **Никакого шума** — убрано всё лишнее

## Принципы "Структурной эстетики"

### Точность (Precision)
Как архитектор выверяет каждый миллиметр, так и мы:
- Отступы кратны 4px
- Радиусы скромные (4-12px)
- Размеры шрифтов по шкале (12, 14, 16, 20, 24, 32)
- Line-height для читабельности, не для красоты

### Модульность (Modularity)
Система, а не коллекция компонентов:
- Переиспользуемые блоки
- Константы вместо магических чисел
- Одинаковая структура везде
- Легко добавлять новое, не ломая старое

### Чистота (Clarity)
Ничего лишнего:
- Минимум цветов (5-7 оттенков серого + 3-4 акцента)
- Минимум размеров шрифта (5-6 вариантов)
- Минимум весов шрифта (400, 500, 600)
- Максимум воздуха вокруг важного

### Контекст (Context)
Дизайн служит контенту:
- Архитектурные элементы — главные
- UI — вспомогательный
- Информация легко считывается
- Действия очевидны

### Уважение (Respect)
К архитектурному наследию и пользователю:
- Не упрощаем до примитива
- Показываем детали, пропорции, формы
- Не инфантилизируем (никаких мультяшек)
- Серьёзное отношение к теме

## Технические решения

### Цветовая палитра "Каменная кладка"

Вдохновлена материалами архитектуры — камень, бетон, металл:

```
Фон:
  - Белый холст: #FFFFFF (чистая бумага)
  - Светлый камень: #F6F8FA (фон приложения)
  
Текст:
  - Чёрный графит: #24292F (основной текст)
  - Серый гранит: #57606A (вторичный текст)
  - Светлый мрамор: #8C959F (неактивное)

Действия:
  - Синий сланец: #0969DA (кликабельное)
  - Тёмный сланец: #0550AE (pressed)
  
Редкость (приглушённая):
  - Обычный: #6E7781 (известняк)
  - Редкий: #0969DA (лазурит)
  - Эпический: #8250DF (аметист)
  - Легендарный: #BF8700 (золото)

Границы:
  - Контур: #D0D7DE (тонкий)
  - Разделитель: #EAEEF2 (ещё тоньше)
```

### Типографика "IBM Plex"

Технический, но гуманный. Как хороший модернизм:
- **Sans** для интерфейса
- **Mono** для цифр (очки, уровни, координаты)
- Меньшие размеры для плотности
- Достаточный line-height для читабельности

### Spacing "Модульная сетка"

Система 4px (как кирпичная кладка):
```
4px  (xs)  — минимальный зазор
8px  (sm)  — между элементами
12px (md)  — между группами
16px (base) — стандартный padding
20px (lg)  — между секциями
24px (xl)  — крупные отступы
```

### Radius "Скромное скругление"

Не круглое, а чуть смягчённое:
```
4px  — мелкие элементы (badges)
6px  — кнопки, inputs (основное)
8px  — карточки
12px — модальные окна
```

## Что НЕ делаем

### Избегаем:
- Яркие градиенты (отвлекают)
- Крупные тени (создают шум)
- Слишком круглые углы (инфантильно)
- Мелкие детали UI (конкурируют с архитектурой)
- Анимации ради анимаций (только функциональные)
- Эмодзи в интерфейсе (серьёзный продукт)
- Декоративные элементы (всё функционально)

### Не копируем:
- GitHub (вдохновляемся структурой, но это не репозиторий)
- Linear (вдохновляемся чистотой, но это не таск-менеджер)
- Другие apps (создаём своё)

## Компоненты

### Карточка элемента
```
┌─────────────────┐
│                 │
│   [Фото 100%]   │ ← Элемент во всей красе
│                 │
├─────────────────┤
│ Метопа          │ ← Название (16px, Medium)
│ [Редкий]        │ ← Бейдж редкости
└─────────────────┘
  ↑ Обводка цветом редкости (2px)
```

### Кнопка
```
┌──────────────┐
│   Действие   │ ← 14px, 500 weight
└──────────────┘
   6px radius
   Высота: 40px (достаточно для тача)
```

### Модальное окно
```
╔═════════════════════════╗
║ Заголовок (20px, 600)   ║
╟─────────────────────────╢
║                         ║
║ Контент (14px, 400)     ║
║                         ║
╚═════════════════════════╝
  ↑ 12px radius сверху
```

## Референсы духа (не копируем, вдохновляемся)

### Архитектура:
- **Баухаус** — форма следует функции
- **Модернизм** — честность материалов
- **Швейцарский стиль** — сетка и типографика
- **Японский минимализм** — пустота как элемент

### Цифровое:
- **Figma** — чистота интерфейса
- **Notion** — функциональность без шума
- **Arc Browser** — современная элегантность
- **Cal.com** — структура и простота

### Печатное:
- **Swiss Design** — модульная сетка
- **Architectural Digest** — уважение к деталям
- **Monocle** — качество без крика

## Реализация

### Все константы в одном месте:
```typescript
// constants/design.ts
export const Colors = { ... }
export const Typography = { ... }
export const Spacing = { ... }
export const BorderRadius = { ... }
```

### Использование:
```typescript
import { Colors, Spacing, Typography } from '@/constants/design';

const styles = StyleSheet.create({
  container: {
    backgroundColor: Colors.surface,
    padding: Spacing.base,
  },
  title: {
    ...Typography.h2,
    color: Colors.textPrimary,
  },
});
```

## Чеклист перед релизом компонента

- [ ] Используются константы из `design.ts`
- [ ] Отступы кратны 4px
- [ ] Размер шрифта из шкалы (12, 14, 16, 20, 24, 32)
- [ ] Цвета из палитры
- [ ] Radius из системы (4, 6, 8, 12)
- [ ] Тач-область минимум 40x40px
- [ ] Контрастность текста достаточна
- [ ] Нет лишних декоративных элементов
- [ ] Компонент следует модульной сетке
